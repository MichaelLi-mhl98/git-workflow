KBQ 1: Explain the Poor/Strong Performance of a Particular Territory
Business Question Explanation
This KBQ seeks to understand why a specific territory is underperforming or outperforming relative to expectations or other territories. The analysis should identify contributing factors from sales activities, marketing engagement, prescriber behavior, and target coverage.
Specific Metrics to Answer the KBQ

Prescription Performance: TRx, NRx trends by territory
Sales Activity Levels: Call frequency, reach, product detailing
Sample Distribution: Volume and coverage of samples distributed
Marketing Engagement: Email open/click rates, webinar attendance
HCP Coverage: % of target HCPs contacted vs. total targets
Activity-to-Outcome Ratios: Calls per Rx, samples per Rx

Data Gaps Preventing Complete Analysis

No incentive compensation data
No RTE (Reach, Frequency, Impact) metrics
No sales collateral/asset usage tracking
No historical call plan data
No financial performance metrics (revenue, costs)

SQL Queries:

KBQ 1.1: Territory Prescription Performance Overview
-- KBQ 1.1: Compare prescription performance across territories
-- Template version
SELECT 
    hsp.cns_prescriber_zipcode,
    LEFT(hsp.cns_prescriber_zipcode, 3) as territory_prefix,
    COUNT(DISTINCT hpa.prescriber_npi) as prescriber_count,
    SUM(hpa.total_trx) as total_trx,
    SUM(hpa.total_units) as total_units,
    SUM(hpa.total_trx) / NULLIF(COUNT(DISTINCT hpa.prescriber_npi), 0) as avg_trx_per_prescriber
FROM hcp_prescribing_activity hpa
JOIN hcp_specialty_zip hsp ON CAST(hpa.prescriber_npi AS CHAR) = CAST(hsp.npi AS CHAR)
WHERE hpa.date_submitted BETWEEN '{start_date}' AND '{end_date}'
    AND hsp.cns_prescriber_zipcode = {territory_zip}
GROUP BY hsp.cns_prescriber_zipcode, LEFT(hsp.cns_prescriber_zipcode, 3)
ORDER BY total_trx DESC;

-- Test version
SELECT 
    hsp.cns_prescriber_zipcode,
    LEFT(hsp.cns_prescriber_zipcode, 3) as territory_prefix,
    COUNT(DISTINCT hpa.prescriber_npi) as prescriber_count,
    SUM(hpa.total_trx) as total_trx,
    SUM(hpa.total_units) as total_units,
    SUM(hpa.total_trx) / NULLIF(COUNT(DISTINCT hpa.prescriber_npi), 0) as avg_trx_per_prescriber
FROM hcp_prescribing_activity hpa
JOIN hcp_specialty_zip hsp ON CAST(hpa.prescriber_npi AS CHAR) = CAST(hsp.npi AS CHAR)
WHERE hpa.date_submitted BETWEEN '2023-01-01' AND '2025-06-30'
    AND hsp.cns_prescriber_zipcode IS NOT NULL
GROUP BY hsp.cns_prescriber_zipcode, LEFT(hsp.cns_prescriber_zipcode, 3)
ORDER BY total_trx DESC
LIMIT 20;

KBQ 1.3: Territory Coverage Analysis
-- KBQ 1.3: Measure HCP target coverage by territory
-- Template version
SELECT 
    h.State as territory,
    COUNT(DISTINCT h.NPI) as total_targets,
    COUNT(DISTINCT s.NPI) as contacted_targets,
    COUNT(DISTINCT s.NPI) / NULLIF(COUNT(DISTINCT h.NPI), 0) * 100 as coverage_percentage,
    COUNT(DISTINCT CASE WHEN h.CNS_HCP_Priority = 'A' THEN h.NPI END) as a_targets,

LEFT JOIN sales_rep_call_activity s ON h.NPI = s.NPI

WHERE h.State = '{territory_state}'
GROUP BY h.State;

KBQ 1.6: Activity-to-Outcome Correlation
-- KBQ 1.6: Calculate key performance ratios for territory efficiency
-- Template version
SELECT 
    sr.State as territory,
    COUNT(DISTINCT sr.Call_ID) as total_calls,
    SUM(hpa.total_trx) as total_trx,
    COUNT(DISTINCT sr.Call_ID) / NULLIF(SUM(hpa.total_trx), 0) as calls_per_rx,
    COUNT(DISTINCT sr.NPI) as hcps_called,
    SUM(hpa.total_trx) / NULLIF(COUNT(DISTINCT sr.NPI), 0) as rx_per_hcp_called,
    COUNT(DISTINCT CASE WHEN sr.Call_Activity_Type = 'Face-to-Face with HCP' THEN sr.Call_ID END) as f2f_calls,
    SUM(CASE WHEN sr.Call_Activity_Type = 'Face-to-Face with HCP' THEN hpa.total_trx ELSE 0 END) / 
        NULLIF(COUNT(DISTINCT CASE WHEN sr.Call_Activity_Type = 'Face-to-Face with HCP' THEN sr.Call_ID END), 0) as rx_per_f2f_call
FROM sales_rep_call_activity sr
LEFT JOIN hcp_prescribing_activity hpa ON sr.NPI = hpa.prescriber_npi
    AND hpa.date_submitted BETWEEN DATE_SUB(sr.Call_Date, INTERVAL 30 DAY) AND DATE_ADD(sr.Call_Date, INTERVAL 30 DAY)
WHERE sr.Call_Date BETWEEN '{start_date}' AND '{end_date}'
    AND sr.State = '{territory_state}'
GROUP BY sr.State;

-- Test version
SELECT 
    sr.State as territory,
    COUNT(DISTINCT sr.Call_ID) as total_calls,
    SUM(hpa.total_trx) as total_trx,
    COUNT(DISTINCT sr.Call_ID) / NULLIF(SUM(hpa.total_trx), 0) as calls_per_rx,
    COUNT(DISTINCT sr.NPI) as hcps_called,
    SUM(hpa.total_trx) / NULLIF(COUNT(DISTINCT sr.NPI), 0) as rx_per_hcp_called,
    COUNT(DISTINCT CASE WHEN sr.Call_Activity_Type = 'Face-to-Face with HCP' THEN sr.Call_ID END) as f2f_calls,
    SUM(CASE WHEN sr.Call_Activity_Type = 'Face-to-Face with HCP' THEN hpa.total_trx ELSE 0 END) / 
        NULLIF(COUNT(DISTINCT CASE WHEN sr.Call_Activity_Type = 'Face-to-Face with HCP' THEN sr.Call_ID END), 0) as rx_per_f2f_call
FROM sales_rep_call_activity sr
LEFT JOIN hcp_prescribing_activity hpa ON sr.NPI = hpa.prescriber_npi
    AND hpa.date_submitted BETWEEN DATE_SUB(sr.Call_Date, INTERVAL 30 DAY) AND DATE_ADD(sr.Call_Date, INTERVAL 30 DAY)
WHERE sr.Call_Date BETWEEN '2023-01-01' AND '2025-06-30'
GROUP BY sr.State
HAVING total_calls > 100
ORDER BY rx_per_hcp_called DESC;

Proxy Metrics and Assumptions

Territory Definition: Using State as primary territory identifier (could also use ZIP code prefixes)
Performance Proxy: Using TRx volume as primary performance indicator (no revenue data)
Activity Impact Window: Assuming 30-day window for call-to-prescription correlation
Coverage Calculation: Based on HCP target list vs. actual contacts
Engagement Metrics: Email open/click rates as proxy for marketing effectiveness

KBQ 2: How Many New Writers Were Generated During Time Period?
Business Question Explanation
This KBQ aims to identify healthcare providers who prescribed the company's products for the first time during a specified period. Understanding new writer acquisition helps evaluate sales and marketing effectiveness in expanding prescriber base.
Specific Metrics to Answer the KBQ

New Writer Count: Total unique prescribers who wrote their first prescription
Product Breakdown: New writers by product (Adzenys, Cotempla, Karbinal)
Geographic Distribution: New writers by state/territory
Conversion Sources: Correlation with sales calls, samples, marketing touches
Time to First Rx: Days from first contact to first prescription

Data Gaps Preventing Complete Analysis

No incentive compensation data
No RTE metrics or sales suggestions data
No media plan execution data
No competitive prescribing data to identify true market-new vs. switch writers
No historical data before 2023 to identify truly new writers

SQL Queries:

KBQ 2.1: Count New Writers by Product
-- KBQ 2.1: Identify new writers who started prescribing during the period
-- Template version
SELECT 
    product_name,
    COUNT(DISTINCT prescriber_npi) as new_writers,
    SUM(first_period_trx) as initial_trx_volume,
    AVG(first_period_trx) as avg_initial_trx
FROM (
    SELECT 
        prescriber_npi,
        product_name,
        MIN(date_submitted) as first_rx_date,
        SUM(CASE WHEN date_submitted BETWEEN '{start_date}' AND '{end_date}' THEN total_trx ELSE 0 END) as first_period_trx
    FROM hcp_prescribing_activity
    GROUP BY prescriber_npi, product_name
    HAVING MIN(date_submitted) BETWEEN '{start_date}' AND '{end_date}'
) new_writers
GROUP BY product_name;

-- Test version
SELECT 
    product_name,
    COUNT(DISTINCT prescriber_npi) as new_writers,
    SUM(first_period_trx) as initial_trx_volume,
    AVG(first_period_trx) as avg_initial_trx
FROM (
    SELECT 
        prescriber_npi,
        product_name,
        MIN(date_submitted) as first_rx_date,
        SUM(CASE WHEN date_submitted BETWEEN '2024-01-01' AND '2024-12-31' THEN total_trx ELSE 0 END) as first_period_trx
    FROM hcp_prescribing_activity
    GROUP BY prescriber_npi, product_name
    HAVING MIN(date_submitted) BETWEEN '2024-01-01' AND '2024-12-31'
) new_writers
GROUP BY product_name;

-- KBQ 2.3: Correlation between sales activities and new writer generation

SELECT 
    activity_type,
    COUNT(DISTINCT prescriber_npi) as new_writers,
    AVG(calls_before_first_rx) as avg_calls_to_convert,
    AVG(days_to_first_rx) as avg_days_to_convert

    HAVING MIN(hpa.date_submitted) BETWEEN '{start_date}' AND '{end_date}'

GROUP BY activity_type;

KBQ 2.4: Sample Impact on New Writers

        HAVING MIN(date_submitted) BETWEEN '{start_date}' AND '{end_date}'
    )
    GROUP BY hpa.prescriber_npi, sampled_flag

GROUP BY sampled_flag;

KBQ 2.5: Marketing Touchpoint Analysis for New Writers
-- KBQ 2.5: Email marketing influence on new writer generation
-- Template version
SELECT 
    email_engagement,
    COUNT(DISTINCT prescriber_npi) as new_writers,
    AVG(emails_before_rx) as avg_emails_before_rx,
    AVG(days_from_first_email) as avg_days_from_first_email
FROM (
    SELECT 
        hpa.prescriber_npi,
        COUNT(DISTINCT e.Campaign_ID) as emails_before_rx,
        DATEDIFF(MIN(hpa.date_submitted), MIN(e.`Created_Date-Time`)) as days_from_first_email,
        CASE 
            WHEN MAX(CASE WHEN e.`Clicked_Date-Time` IS NOT NULL THEN 1 ELSE 0 END) = 1 THEN 'Clicked'
            WHEN MAX(CASE WHEN e.`Opened_Date-Time` IS NOT NULL THEN 1 ELSE 0 END) = 1 THEN 'Opened'
            WHEN COUNT(e.HCP_NPI) > 0 THEN 'Received'
            ELSE 'No_Email'
        END as email_engagement
    FROM hcp_prescribing_activity hpa
    LEFT JOIN email_activity e ON hpa.prescriber_npi = e.HCP_NPI
        AND e.`Created_Date-Time` <= hpa.date_submitted
    WHERE hpa.prescriber_npi IN (
        SELECT prescriber_npi 
        FROM hcp_prescribing_activity 
        GROUP BY prescriber_npi 
        HAVING MIN(date_submitted) BETWEEN '{start_date}' AND '{end_date}'
    )
    GROUP BY hpa.prescriber_npi
) email_analysis
GROUP BY email_engagement;

KBQ 2.6: Monthly New Writer Trend
-- KBQ 2.6: Track new writer acquisition trend over time
-- Template version
SELECT 
    DATE_FORMAT(first_rx_date, '%Y-%m') as month,
    COUNT(DISTINCT prescriber_npi) as new_writers,
    SUM(month_trx) as new_writer_trx,
    AVG(month_trx) as avg_trx_per_new_writer
FROM (
    SELECT 
        prescriber_npi,
        MIN(date_submitted) as first_rx_date,
        SUM(total_trx) as month_trx
    FROM hcp_prescribing_activity
    WHERE date_submitted BETWEEN '{start_date}' AND '{end_date}'
    GROUP BY prescriber_npi
    HAVING MIN(date_submitted) BETWEEN '{start_date}' AND '{end_date}'
) monthly_new
GROUP BY DATE_FORMAT(first_rx_date, '%Y-%m')
ORDER BY month;

KBQ 3.3: Sales Coverage Analysis for Lapsed Writers
-- KBQ 3.3: Examine sales activity patterns before prescriber lapse

    coverage_status,

        calls_before_lapse,
        DATEDIFF(lapsed.last_rx_date, calls.last_call_date) as days_between_last_call_and_rx

GROUP BY coverage_status;

KBQ 3.4: Sample and Marketing Gap Analysis
-- KBQ 3.4: Identify sample/marketing gaps for declining prescribers
-- Template version
SELECT 
    prescriber_category,
    COUNT(DISTINCT prescriber_npi) as prescriber_count,
    AVG(sample_gap_days) as avg_days_since_last_sample,
    AVG(email_gap_days) as avg_days_since_last_email,
    AVG(trx_decline_pct) as avg_prescription_decline
FROM (
    SELECT 
        d.prescriber_npi,
        d.trx_decline_pct,
        DATEDIFF('{end_date}', MAX(sa.Sample_Date)) as sample_gap_days,
        DATEDIFF('{end_date}', MAX(e.`Created_Date-Time`)) as email_gap_days,
        CASE 
            WHEN d.trx_decline_pct < -50 THEN 'Severe_Decline'
            WHEN d.trx_decline_pct < -25 THEN 'Moderate_Decline'
            ELSE 'Mild_Decline'
        END as prescriber_category
    FROM (
        SELECT 
            prescriber_npi,
            (SUM(CASE WHEN date_submitted >= DATE_SUB('{end_date}', INTERVAL 6 MONTH) THEN total_trx ELSE 0 END) -
             SUM(CASE WHEN date_submitted < DATE_SUB('{end_date}', INTERVAL 6 MONTH) AND date_submitted >= DATE_SUB('{end_date}', INTERVAL 12 MONTH) THEN total_trx ELSE 0 END)) /
            NULLIF(SUM(CASE WHEN date_submitted < DATE_SUB('{end_date}', INTERVAL 6 MONTH) AND date_submitted >= DATE_SUB('{end_date}', INTERVAL 12 MONTH) THEN total_trx ELSE 0 END), 0) * 100 as trx_decline_pct
        FROM hcp_prescribing_activity
        WHERE date_submitted BETWEEN DATE_SUB('{end_date}', INTERVAL 12 MONTH) AND '{end_date}'
        GROUP BY prescriber_npi
        HAVING SUM(CASE WHEN date_submitted < DATE_SUB('{end_date}', INTERVAL 6 MONTH) THEN total_trx ELSE 0 END) > 0

    LEFT JOIN email_activity e ON d.prescriber_npi = e.HCP_NPI
    GROUP BY d.prescriber_npi, d.trx_decline_pct, prescriber_category

KBQ 4.2: Seasonal Diagnosis Patterns
-- KBQ 4.2: Identify seasonal patterns in new patient starts

        DATE_FORMAT(date_submitted, '%Y-%m') as year_month,
        MONTH(date_submitted) as month_num,
        MONTHNAME(date_submitted) as month_name,
        SUM(nrx) as monthly_nrx
    FROM trend_data
    WHERE `year_month` BETWEEN '{start_date}' AND '{end_date}'
        AND nrx IS NOT NULL
    GROUP BY DATE_FORMAT(date_submitted, '%Y-%m'), MONTH(date_submitted), MONTHNAME(date_submitted)
) monthly_data
GROUP BY MONTH(date_submitted), MONTHNAME(date_submitted)
ORDER BY month_num;

KBQ 5: Predict Existing Patient Discontinuations
Business Question Explanation
This KBQ aims to forecast patient discontinuation risk by analyzing prescription patterns, refill rates, and prescriber engagement. Without patient-level data, we use declining prescription volumes and refill patterns (TRx minus NRx) as proxies for patient persistence and discontinuation risk.
Specific Metrics to Answer the KBQ

Refill Rates: TRx vs NRx ratios indicating patient retention
Prescription Gaps: Time between prescriptions suggesting discontinuation
Volume Declines: Prescriber-level drops indicating patient loss
Engagement Correlation: Sales/marketing activity impact on persistence
Historical Patterns: Seasonal or temporal discontinuation trends

Data Gaps Preventing Complete Analysis

No patient-level prescription or refill data
No adherence or compliance metrics
No patient outcome or satisfaction data
No insurance coverage changes
No adverse event or side effect data

KBQ 5.6: Discontinuation Risk Scoring
-- KBQ 5.6: Create composite discontinuation risk score for prediction
-- Template version
SELECT 
    risk_tier,
    COUNT(DISTINCT prescriber_npi) as prescriber_count,
    SUM(current_trx) as total_rx_at_risk,
    AVG(risk_score) as avg_risk_score,
    SUM(current_trx) * AVG(discontinuation_probability) as predicted_discontinued_rx
FROM (
    SELECT 
        prescriber_npi,
        current_trx,
        volume_decline_score + gap_score + engagement_score as risk_score,
        CASE 
            WHEN volume_decline_score + gap_score + engagement_score >= 7 THEN 'High_Risk'
            WHEN volume_decline_score + gap_score + engagement_score >= 4 THEN 'Medium_Risk'
            ELSE 'Low_Risk'
        END as risk_tier,
        (volume_decline_score + gap_score + engagement_score) / 9.0 as discontinuation_probability
    FROM (
        SELECT 
            hpa.prescriber_npi,
            SUM(CASE WHEN hpa.date_submitted >= DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN hpa.total_trx ELSE 0 END) as current_trx,
            CASE 
                WHEN (SUM(CASE WHEN hpa.date_submitted >= DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN hpa.total_trx ELSE 0 END) /
                      NULLIF(SUM(CASE WHEN hpa.date_submitted < DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN hpa.total_trx ELSE 0 END), 0)) < 0.5 THEN 3
                WHEN (SUM(CASE WHEN hpa.date_submitted >= DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN hpa.total_trx ELSE 0 END) /
                      NULLIF(SUM(CASE WHEN hpa.date_submitted < DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN hpa.total_trx ELSE 0 END), 0)) < 0.75 THEN 2
                ELSE 1
            END as volume_decline_score,
            CASE 
                WHEN MAX(DATEDIFF('{end_date}', hpa.date_submitted)) > 90 THEN 3
                WHEN MAX(DATEDIFF('{end_date}', hpa.date_submitted)) > 60 THEN 2
                ELSE 1
            END as gap_score,
            CASE 
                WHEN COUNT(DISTINCT sr.Call_ID) = 0 THEN 3
                WHEN COUNT(DISTINCT sr.Call_ID) < 3 THEN 2
                ELSE 1
            END as engagement_score
        FROM hcp_prescribing_activity hpa

        SUM(hpa.total_trx) as total_rx,
        SUM(hpa.total_trx) / NULLIF(COUNT(DISTINCT sr.Call_ID), 0) as rx_per_call,
        COUNT(DISTINCT CASE WHEN first_rx.first_rx_date >= '{start_date}' THEN hpa.prescriber_npi END) as new_writers,
        COUNT(DISTINCT sr.NPI) * 100.0 / NULLIF(COUNT(DISTINCT h.NPI), 0) as hcp_coverage,
        RANK() OVER (ORDER BY SUM(hpa.total_trx) DESC) as performance_rank,
        RANK() OVER (ORDER BY SUM(hpa.total_trx) / NULLIF(COUNT(DISTINCT sr.Call_ID), 0) DESC) as efficiency_rank,
        RANK() OVER (ORDER BY COUNT(DISTINCT CASE WHEN first_rx.first_rx_date >= '{start_date}' THEN hpa.prescriber_npi END) DESC) as growth_rank
    FROM sales_rep_call_activity sr
    LEFT JOIN hcp_list h ON sr.State = h.State
    LEFT JOIN hcp_prescribing_activity hpa ON sr.NPI = hpa.prescriber_npi
        AND hpa.date_submitted BETWEEN '{start_date}' AND '{end_date}'
    LEFT JOIN (
        SELECT prescriber_npi, MIN(date_submitted) as first_rx_date
        FROM hcp_prescribing_activity
        GROUP BY prescriber_npi
    ) first_rx ON hpa.prescriber_npi = first_rx.prescriber_npi
    WHERE sr.Call_Date BETWEEN '{start_date}' AND '{end_date}'
    GROUP BY sr.State
) performance_metrics;


SELECT 
    State as territory,
    total_rx,
    rx_per_call,
    new_writers,
    calls_made,
    (performance_rank + efficiency_rank + growth_rank) / 3.0 as composite_score,
    CASE 
        WHEN (performance_rank + efficiency_rank + growth_rank) / 3.0 <= 5 THEN 'Top_Performer'
        WHEN (performance_rank + efficiency_rank + growth_rank) / 3.0 >= 15 THEN 'Bottom_Performer'
        ELSE 'Middle_Performer'
    END as performance_tier
FROM (
    SELECT 
        sr.State,
        COUNT(DISTINCT sr.Call_ID) as calls_made,
        COUNT(DISTINCT sr.NPI) as hcps_contacted,
        SUM(rx.total_rx) as total_rx,
        SUM(rx.total_rx) / NULLIF(COUNT(DISTINCT sr.Call_ID), 0) as rx_per_call,
        SUM(rx.new_writer) as new_writers,
        RANK() OVER (ORDER BY SUM(rx.total_rx) DESC) as performance_rank,
        RANK() OVER (ORDER BY SUM(rx.total_rx) / NULLIF(COUNT(DISTINCT sr.Call_ID), 0) DESC) as efficiency_rank,
        RANK() OVER (ORDER BY SUM(rx.new_writer) DESC) as growth_rank
    FROM sales_rep_call_activity sr
    LEFT JOIN (
        SELECT 
            hpa.prescriber_npi,
            SUM(hpa.total_trx) as total_rx,
            MAX(CASE WHEN hpa.date_submitted = first_rx.first_rx_date 
                     AND first_rx.first_rx_date >= '2024-01-01' 
                THEN 1 ELSE 0 END) as new_writer
        FROM hcp_prescribing_activity hpa
        JOIN (
            SELECT prescriber_npi, MIN(date_submitted) as first_rx_date
            FROM hcp_prescribing_activity
            GROUP BY prescriber_npi
        ) first_rx ON hpa.prescriber_npi = first_rx.prescriber_npi
        WHERE hpa.date_submitted BETWEEN '2024-01-01' AND '2025-06-30'
        GROUP BY hpa.prescriber_npi
    ) rx ON sr.NPI = rx.prescriber_npi
    WHERE sr.Call_Date BETWEEN '2024-01-01' AND '2025-06-30'
    GROUP BY sr.State
    HAVING COUNT(DISTINCT sr.Call_ID) > 50
) performance_metrics
ORDER BY composite_score
LIMIT 20;

KBQ 10.4: Timing Preferences by Segment
-- KBQ 10.4: Identify when different segments are most responsive
-- Template version
SELECT 
    customer_segment,
    call_day_preference,
    email_day_preference,
    preferred_call_time,
    webinar_attendance_pattern,
    avg_response_time_hours,
    total_segment_rx
FROM (
    SELECT 
        h.Specialty as customer_segment,
        MODE() WITHIN GROUP (ORDER BY DAYNAME(sr.Call_Date)) as call_day_preference,
        MODE() WITHIN GROUP (ORDER BY DAYNAME(e.`Created_Date-Time`)) as email_day_preference,
        CASE 
            WHEN AVG(HOUR(sr.Call_Date)) < 12 THEN 'Morning'
            WHEN AVG(HOUR(sr.Call_Date)) < 17 THEN 'Afternoon'
            ELSE 'Evening'
        END as preferred_call_time,
        COUNT(DISTINCT w.Date_of_Webinar) as webinar_attendance_pattern,
        AVG(TIMESTAMPDIFF(HOUR, e.`Created_Date-Time`, e.`Opened_Date-Time`)) as avg_response_time_hours,
        SUM(hpa.total_rx) as total_segment_rx
    FROM hcp_list h
    LEFT JOIN sales_rep_call_activity sr ON h.NPI = sr.NPI
        AND sr.Call_Date BETWEEN '{start_date}' AND '{end_date}'
    LEFT JOIN email_activity e ON h.NPI = e.HCP_NPI
        AND e.`Created_Date-Time` BETWEEN '{start_date}' AND '{end_date}'

KBQ 10.6: Segment Engagement Drivers Analysis
-- KBQ 10.6: Identify key factors driving engagement differences
-- Template version
SELECT 
    high_engagement_segment,
    low_engagement_segment,
    factor_name,
    high_segment_value,
    low_segment_value,
    difference,
    impact_score
FROM (
    SELECT 
        'High_Engaged' as high_engagement_segment,
        'Low_Engaged' as low_engagement_segment,
        'Avg_Call_Frequency' as factor_name,
        AVG(CASE WHEN engagement_tier = 'High' THEN call_frequency END) as high_segment_value,
        AVG(CASE WHEN engagement_tier = 'Low' THEN call_frequency END) as low_segment_value,
        AVG(CASE WHEN engagement_tier = 'High' THEN call_frequency END) - 
            AVG(CASE WHEN engagement_tier = 'Low' THEN call_frequency END) as difference,
        ABS(AVG(CASE WHEN engagement_tier = 'High' THEN call_frequency END) - 
            AVG(CASE WHEN engagement_tier = 'Low' THEN call_frequency END)) / 
            NULLIF(AVG(call_frequency), 0) * 100 as impact_score
    FROM (
        SELECT 
            h.NPI,
            CASE 
                WHEN (call_count + email_count + webinar_count) >= 10 THEN 'High'
                WHEN (call_count + email_count + webinar_count) <= 2 THEN 'Low'
                ELSE 'Medium'
            END as engagement_tier,
            COALESCE(call_count, 0) as call_frequency,
            COALESCE(email_open_rate, 0) as email_responsiveness,
            COALESCE(sample_count, 0) as sample_exposure,
            COALESCE(total_rx, 0) as prescription_volume
        FROM hcp_list h
        LEFT JOIN (
            SELECT NPI, COUNT(DISTINCT Call_ID) as call_count
            FROM sales_rep_call_activity
            WHERE Call_Date BETWEEN '{start_date}' AND '{end_date}'
            GROUP BY NPI
        ) calls ON h.NPI = calls.NPI
        LEFT JOIN (
            SELECT HCP_NPI, 
                COUNT(*) as email_count,
                COUNT(CASE WHEN `Opened_Date-Time` IS NOT NULL THEN 1 END) * 100.0 / NULLIF(COUNT(*), 0) as email_open_rate
            FROM email_activity
            WHERE `Created_Date-Time` BETWEEN '{start_date}' AND '{end_date}'
            GROUP BY HCP_NPI
        ) emails ON h.NPI = emails.HCP_NPI
        LEFT JOIN (
            SELECT npi, COUNT(*) as webinar_count
            FROM webinar
            WHERE Date_of_Webinar BETWEEN '{start_date}' AND '{end_date}'
            GROUP BY npi
        ) webinars ON h.NPI = webinars.npi
        LEFT JOIN (
            SELECT sr.NPI, SUM(sa.Quantity) as sample_count
            FROM sales_rep_call_activity sr
            JOIN sample_activity sa ON sr.Call_ID = sa.Call_ID
            WHERE sr.Call_Date BETWEEN '{start_date}' AND '{end_date}'
            GROUP BY sr.NPI
        ) samples ON h.NPI = samples.NPI
        LEFT JOIN (
            SELECT prescriber_npi, SUM(total_trx) as total_rx
            FROM hcp_prescribing_activity
            WHERE date_submitted BETWEEN '{start_date}' AND '{end_date}'
            GROUP BY prescriber_npi
        ) rx ON h.NPI = rx.prescriber_npi
    ) engagement_analysis
    WHERE engagement_tier IN ('High', 'Low')
) driver_comparison;

KBQ 11.1: Identify High-Opportunity Targets
-- KBQ 11.1: Score HCPs for targeting priority
-- Template version
SELECT 
    h.NPI,
    h.Last_Name,
    h.Specialty,
    h.State,
    (COALESCE(rx_growth_score, 0) + COALESCE(engagement_score, 0) + COALESCE(recency_score, 0)) as opportunity_score,
    CASE 
        WHEN days_since_contact > 60 THEN 'Immediate_Contact'
        WHEN rx_trend > 0 AND days_since_contact > 30 THEN 'Follow_Up'
        WHEN new_writer = 1 THEN 'Nurture_New'
        ELSE 'Maintain'
    END as recommended_action
FROM hcp_list h
LEFT JOIN (
    SELECT 
        prescriber_npi,
        SUM(CASE WHEN date_submitted >= DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN total_trx ELSE 0 END) -
        SUM(CASE WHEN date_submitted < DATE_SUB('{end_date}', INTERVAL 3 MONTH) THEN total_trx ELSE 0 END) as rx_trend,
        CASE WHEN MIN(date_submitted) >= DATE_SUB('{end_date}', INTERVAL 6 MONTH) THEN 1 ELSE 0 END as new_writer,
        (SUM(total_trx) / 10) as rx_growth_score
    FROM hcp_prescribing_activity
    WHERE date_submitted BETWEEN DATE_SUB('{end_date}', INTERVAL 6 MONTH) AND '{end_date}'
    GROUP BY prescriber_npi
) rx ON h.NPI = rx.prescriber_npi
LEFT JOIN (
    SELECT 
        NPI,
        DATEDIFF('{end_date}', MAX(Call_Date)) as days_since_contact,
        COUNT(DISTINCT Call_ID) * 10 as engagement_score
    FROM sales_rep_call_activity
    WHERE Call_Date BETWEEN DATE_SUB('{end_date}', INTERVAL 6 MONTH) AND '{end_date}'
    GROUP BY NPI
) engage ON h.NPI = engage.NPI
CROSS JOIN (SELECT CASE WHEN days_since_contact <= 30 THEN 20 ELSE 0 END as recency_score) scoring
WHERE h.State = '{state}'
ORDER BY opportunity_score DESC
LIMIT 100;

KBQ 11.2: Channel Recommendation Engine
-- KBQ 11.2: Recommend optimal channel for each HCP
-- Template version
SELECT 
    NPI,
    best_channel,
    channel_score,
    last_channel_used,
    days_since_last_touch,
    CASE 
        WHEN best_channel != last_channel_used THEN CONCAT('Switch to ', best_channel)
        WHEN days_since_last_touch > 45 THEN CONCAT('Re-engage via ', best_channel)
        ELSE CONCAT('Continue ', best_channel)
    END as channel_recommendation
FROM (
    SELECT 
        h.NPI,
        CASE 
            WHEN call_effectiveness > email_effectiveness AND call_effectiveness > webinar_effectiveness THEN 'Face-to-Face'
            WHEN email_effectiveness > webinar_effectiveness THEN 'Email'
            ELSE 'Webinar'
        END as best_channel,
        GREATEST(call_effectiveness, email_effectiveness, webinar_effectiveness) as channel_score,
        last_channel_used,
        days_since_last_touch
    FROM hcp_list h
    LEFT JOIN (
        SELECT 
            NPI,
            SUM(post_call_rx) / NULLIF(COUNT(DISTINCT Call_ID), 0) as call_effectiveness,
            MAX(Call_Date) as last_call
        FROM sales_rep_call_activity sr
        LEFT JOIN (
            SELECT prescriber_npi, SUM(total_trx) as post_call_rx
            FROM hcp_prescribing_activity
            WHERE date_submitted BETWEEN '{start_date}' AND '{end_date}'
            GROUP BY prescriber_npi
        ) rx ON sr.NPI = rx.prescriber_npi
        WHERE Call_Date BETWEEN '{start_date}' AND '{end_date}'
        GROUP BY NPI
    ) calls ON h.NPI = calls.NPI
    LEFT JOIN (
        SELECT 
            HCP_NPI,
            COUNT(CASE WHEN `Clicked_Date-Time` IS NOT NULL THEN 1 END) * 10 as email_effectiveness,
            MAX(`Created_Date-Time`) as last_email
        FROM email_activity
        WHERE `Created_Date-Time` BETWEEN '{start_date}' AND '{end_date}'
        GROUP BY HCP_NPI
    ) emails ON h.NPI = emails.HCP_NPI
    LEFT JOIN (
        SELECT npi, COUNT(*) * 15 as webinar_effectiveness, MAX(Date_of_Webinar) as last_webinar
        FROM webinar
        WHERE Date_of_Webinar BETWEEN '{start_date}' AND '{end_date}'
        GROUP BY npi
    ) webinars ON h.NPI = webinars.npi
    CROSS JOIN LATERAL (
        SELECT 
            CASE 
                WHEN last_call >= last_email AND last_call >= last_webinar THEN 'Call'
                WHEN last_email >= last_webinar THEN 'Email'
                ELSE 'Webinar'
            END as last_channel_used,
            DATEDIFF('{end_date}', GREATEST(last_call, last_email, last_webinar)) as days_since_last_touch
    ) recent
) channel_analysis
WHERE NPI IN ({target_npis});

KBQ 11.4: Cadence Optimization
-- KBQ 11.4: Determine optimal contact frequency
-- Template version
SELECT 
    NPI,
    historical_response_pattern,
    optimal_frequency_days,
    last_contact_days_ago,
    CASE 
        WHEN last_contact_days_ago >= optimal_frequency_days THEN 'Contact_Now'
        WHEN last_contact_days_ago >= optimal_frequency_days * 0.8 THEN 'Schedule_Soon'
        ELSE CONCAT('Wait_', optimal_frequency_days - last_contact_days_ago, '_Days')
    END as cadence_recommendation
FROM (
    SELECT 
        h.NPI,
        AVG(days_between_contacts) as historical_response_pattern,
        CASE 
            WHEN AVG(response_rate) > 0.5 THEN 14
            WHEN AVG(response_rate) > 0.3 THEN 21
            WHEN AVG(response_rate) > 0.1 THEN 30
            ELSE 45
        END as optimal_frequency_days,
        DATEDIFF('{end_date}', MAX(contact_date)) as last_contact_days_ago
    FROM hcp_list h
    LEFT JOIN (
        SELECT 
            NPI,
            contact_date,
            LAG(contact_date) OVER (PARTITION BY NPI ORDER BY contact_date) as prev_contact,
            DATEDIFF(contact_date, LAG(contact_date) OVER (PARTITION BY NPI ORDER BY contact_date)) as days_between_contacts,
            response_flag / total_contacts as response_rate
        FROM (
            SELECT NPI, Call_Date as contact_date, 1 as response_flag, 1 as total_contacts
            FROM sales_rep_call_activity
            WHERE Call_Date BETWEEN DATE_SUB('{end_date}', INTERVAL 6 MONTH) AND '{end_date}'
            UNION ALL
            SELECT HCP_NPI, DATE(`Created_Date-Time`), 
                CASE WHEN `Opened_Date-Time` IS NOT NULL THEN 1 ELSE 0 END, 1
            FROM email_activity
            WHERE `Created_Date-Time` BETWEEN DATE_SUB('{end_date}', INTERVAL 6 MONTH) AND '{end_date}'
        ) all_contacts
    ) contact_history ON h.NPI = contact_history.NPI
    WHERE h.CNS_HCP_Priority IN ('A', 'B')
    GROUP BY h.NPI
) cadence_analysis
WHERE last_contact_days_ago IS NOT NULL;

        SELECT prescriber_npi,

    -- Additional logic for product gaps and growth potential

-- KBQ 12.1: Analyze conversion rates by follow-up timing
-- Template version
SELECT 
    follow_up_window,
    COUNT(DISTINCT NPI) as prospects,
    SUM(converted) as conversions,
    SUM(converted) * 100.0 / COUNT(*) as conversion_rate,
    AVG(days_to_first_rx) as avg_days_to_rx
FROM (
    SELECT 
        c1.NPI,
        DATEDIFF(c2.Call_Date, c1.Call_Date) as days_to_followup,
        CASE 
            WHEN DATEDIFF(c2.Call_Date, c1.Call_Date) <= 7 THEN '0-7_days'
            WHEN DATEDIFF(c2.Call_Date, c1.Call_Date) <= 14 THEN '8-14_days'
            WHEN DATEDIFF(c2.Call_Date, c1.Call_Date) <= 30 THEN '15-30_days'
            WHEN DATEDIFF(c2.Call_Date, c1.Call_Date) <= 60 THEN '31-60_days'
            ELSE '60+_days'
        END as follow_up_window,
        CASE WHEN rx.first_rx_date IS NOT NULL THEN 1 ELSE 0 END as converted,
        DATEDIFF(rx.first_rx_date, c1.Call_Date) as days_to_first_rx
    FROM (
        SELECT NPI, MIN(Call_Date) as Call_Date
        FROM sales_rep_call_activity
        WHERE Call_Date BETWEEN '{start_date}' AND '{end_date}'
        GROUP BY NPI
    ) c1
    JOIN sales_rep_call_activity c2 ON c1.NPI = c2.NPI AND c2.Call_Date > c1.Call_Date
    LEFT JOIN (
        SELECT prescriber_npi, MIN(date_submitted) as first_rx_date
        FROM hcp_prescribing_activity
        WHERE date_submitted BETWEEN '{start_date}' AND DATE_ADD('{end_date}', INTERVAL 90 DAY)
        GROUP BY prescriber_npi
    ) rx ON c1.NPI = rx.prescriber_npi AND rx.first_rx_date >= c1.Call_Date
) timing_analysis
GROUP BY follow_up_window
ORDER BY conversion_rate DESC;

KBQ 12.4: First Prescription Timing Patterns
-- KBQ 12.4: Analyze time from contact to first prescription
-- Template version
SELECT 
    contact_to_rx_window,
    COUNT(*) as converters,
    AVG(total_contacts_before_rx) as avg_touches_needed,
    AVG(days_to_convert) as avg_conversion_days,
    percentile_25,
    percentile_50,
    percentile_75
FROM (
    SELECT 
        NPI,
        DATEDIFF(first_rx_date, first_contact_date) as days_to_convert,
        total_contacts_before_rx,
        CASE 
            WHEN DATEDIFF(first_rx_date, first_contact_date) <= 30 THEN '0-30_days'
            WHEN DATEDIFF(first_rx_date, first_contact_date) <= 60 THEN '31-60_days'
            WHEN DATEDIFF(first_rx_date, first_contact_date) <= 90 THEN '61-90_days'
            ELSE '90+_days'
        END as contact_to_rx_window,
        PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY DATEDIFF(first_rx_date, first_contact_date)) OVER () as percentile_25,
        PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY DATEDIFF(first_rx_date, first_contact_date)) OVER () as percentile_50,
        PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY DATEDIFF(first_rx_date, first_contact_date)) OVER () as percentile_75
    FROM (
        SELECT 
            rx.prescriber_npi as NPI,
            MIN(rx.date_submitted) as first_rx_date,
            MIN(c.Call_Date) as first_contact_date,
            COUNT(DISTINCT c.Call_ID) as total_contacts_before_rx
        FROM hcp_prescribing_activity rx
        JOIN sales_rep_call_activity c ON rx.prescriber_npi = c.NPI
            AND c.Call_Date <= rx.date_submitted
        WHERE rx.date_submitted BETWEEN '{start_date}' AND '{end_date}'
            AND c.Call_Date >= DATE_SUB(rx.date_submitted, INTERVAL 180 DAY)
        GROUP BY rx.prescriber_npi
        HAVING MIN(c.Call_Date) < MIN(rx.date_submitted)
    ) conversion_timing
) analysis
GROUP BY contact_to_rx_window, percentile_25, percentile_50, percentile_75
ORDER BY contact_to_rx_window;

        best_performing_window,
        CASE 
            WHEN best_performing_window = '0-7_days' THEN 7
            WHEN best_performing_window = '8-14_days' THEN 14
            WHEN best_performing_window = '15-30_days' THEN 21
            ELSE 45
        END as optimal_first_followup_days,
        max_conversion_rate as expected_conversion_rate,
        current_avg_days as current_avg_followup_days,
        max_conversion_rate - current_conversion_rate as improvement_opportunity
    FROM (
        SELECT 
            h.Specialty,
            h.CNS_HCP_Priority,
            AVG(CASE WHEN follow_up_window = '0-7_days' THEN conversion_rate END) as conv_0_7,
            AVG(CASE WHEN follow_up_window = '8-14_days' THEN conversion_rate END) as conv_8_14,
            AVG(CASE WHEN follow_up_window = '15-30_days' THEN conversion_rate END) as conv_15_30,
            AVG(current_days) as current_avg_days,
            AVG(current_rate) as current_conversion_rate,
            GREATEST(
                AVG(CASE WHEN follow_up_window = '0-7_days' THEN conversion_rate END),
                AVG(CASE WHEN follow_up_window = '8-14_days' THEN conversion_rate END),
                AVG(CASE WHEN follow_up_window = '15-30_days' THEN conversion_rate END)
            ) as max_conversion_rate,
            CASE 
                WHEN GREATEST(conv_0_7, conv_8_14, conv_15_30) = conv_0_7 THEN '0-7_days'
                WHEN GREATEST(conv_0_7, conv_8_14, conv_15_30) = conv_8_14 THEN '8-14_days'
                ELSE '15-30_days'
            END as best_performing_window
        FROM hcp_list h
        JOIN (
            -- Analysis subquery here
            SELECT NPI, follow_up_window, conversion_rate, current_days, current_rate
            FROM followup_analysis
        ) fa ON h.NPI = fa.NPI
        GROUP BY h.Specialty, h.CNS_HCP_Priority
    ) optimization
) recommendations
WHERE improvement_opportunity > 5
ORDER BY improvement_opportunity DESC;

    LEFT JOIN (
        SELECT NPI, 1 as prev_call
        FROM sales_rep_call_activity
        WHERE Call_Date BETWEEN '2024-10-01' AND '2024-12-31'
        GROUP BY NPI
    ) prev_call ON e.HCP_NPI = prev_call.NPI
    WHERE e.`Created_Date-Time` BETWEEN '2025-01-01' AND '2025-06-30'
) context_impact
GROUP BY follow_up_type;


KBQ 14.5: Success Factor Summary
-- KBQ 14.5: Identify key success factors for high-performing content
-- Template version
SELECT 
    success_factor,
    impact_on_engagement,
    example_content,
    recommendation
FROM (
    SELECT 'Targeted_Audience' as success_factor,
           AVG(CASE WHEN priority_focus > 0.5 THEN open_rate END) - 
           AVG(CASE WHEN priority_focus <= 0.5 THEN open_rate END) as impact_on_engagement,
           MAX(CASE WHEN priority_focus > 0.5 THEN Template_Name END) as example_content,
           'Focus on high-priority HCPs' as recommendation
    FROM (
        SELECT e.Template_Name,
               COUNT(CASE WHEN h.CNS_HCP_Priority = 'A' THEN 1 END) * 1.0 / COUNT(*) as priority_focus,
               COUNT(CASE WHEN e.`Opened_Date-Time` IS NOT NULL THEN 1 END) * 100.0 / COUNT(*) as open_rate
        FROM email_activity e
        JOIN hcp_list h ON e.HCP_NPI = h.NPI
        WHERE e.`Created_Date-Time` BETWEEN '{start_date}' AND '{end_date}'
        GROUP BY e.Template_Name
    ) targeting
    
    UNION ALL
    
    SELECT 'Clinical_Content',
           AVG(CASE WHEN content_type = 'Clinical' THEN engagement END) - 
           AVG(CASE WHEN content_type != 'Clinical' THEN engagement END),
           'Dosing Guidelines',
           'Use clinical/educational content'
    FROM (
        SELECT CASE WHEN Email_Subject LIKE '%Dosing%' THEN 'Clinical' ELSE 'Other' END as content_type,
               CASE WHEN `Opened_Date-Time` IS NOT NULL THEN 100 ELSE 0 END as engagement
        FROM email_activity
        WHERE `Created_Date-Time` BETWEEN '{start_date}' AND '{end_date}'
    ) content
) success_factors
ORDER BY impact_on_engagement DESC;
